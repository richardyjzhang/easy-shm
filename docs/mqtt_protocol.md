# MQTT 数据上报协议

## 连接信息

| 参数      | 说明                          |
| --------- | ----------------------------- |
| Broker    | 由部署环境决定                |
| 端口      | 1883（TCP） / 8883（TLS）     |
| Client ID | 建议使用 `edge_{structureId}` |
| QoS       | 1（至少一次）                 |

## 主题

```
easy-shm/{structureId}
```

`structureId` 为结构物 ID，对应 MySQL `structure.id`。

示例：`easy-shm/1001`

## 负载格式

JSON 数组，每个元素为一条采集记录：

```json
[
  {
    "ts": "2026-04-05 21:23:15.123",
    "point_id": 10001,
    "val": 23.56
  },
  {
    "ts": "2026-04-05 21:23:15.123",
    "point_id": 10002,
    "val": 0.0037
  }
]
```

### 字段说明

| 字段       | 类型   | 必填 | 说明                                   |
| ---------- | ------ | ---- | -------------------------------------- |
| `ts`       | string | 是   | 采集时间，格式 `YYYY-MM-DD HH:MM:SS.mmm`，使用本地时间 |
| `point_id` | number | 是   | 测点 ID，对应 MySQL `monitor_point.id` |
| `val`      | number | 是   | 采集值（浮点数）                       |

## 约束与注意事项

- 单条消息负载大小建议不超过 **1 MB**
- 单条消息中的所有记录应属于同一个结构物（与主题中的 `structureId` 一致）
- `ts` 使用边缘侧本地时间，服务端不做时区转换
